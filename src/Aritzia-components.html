Category Page

The category page was automatically created by fetching all category information from SFCC and populating the page information that will be discussed below.

Slots

The slots on the Category (PLP) page are localized so that content can be unique across the defined locales (see below). If a localized slot is empty there is no fallback content available and no content will be displayed for that locale.

The slot content types are stored within the slots repos that have been defined as seen in the image below.

Slot repository

There are 3 slot areas available on the Category page.

  • Category Banner Slot - Allows for multiple pieces of content to be put at the top of the page.

  • In-Grid Promo Slot - This controls the content that will be displayed within the grid at the configured locations for Category pages with content.

  • Category Bottom Slot - This slot allows for multiple pieces of content that is displayed at the bottom of the page underneath the products.

Slots

The homepage allows for up to ten slots on the page. Within each slot the editor can define the default content and variations that are associated with customer groups. The slots are also localized, which allows the editor to target specific content to a particular site within the same page as the US site. The diagram below shows the available content types per slot.

The content that is available within the slots is managed through a partial schema with the schema ID slot-available-content.json.

The slot-available-content.json partial schema is where the editor will maintain the available content types for the slots.

Variations

The variations are created within a content type called Home Content Slot Variation or Home Hero Slot Variation, which needs to be created before they can be added to the slot. To build content that is associated with customer groups:

  1. Navigate to the "Content" repository

  2. Click on the homepage folder.

  3. Click on "Create content" and select Home Content Slot Variation or Home Hero Slot Variation depending upon which slot you are adding variations to on the homepage.

Navigation

The navigation is managed through the Amplience Hierarchy structure. It allows for adding top level content items that are lists or are links to pages. The list can be nested to have additional lists or list items within.

The navigation content is stored in a repo called Menu. All available content types that are used to build the top navigation and footer navigation are stored in folders within the Menu repo.

Figure 11. Example navigation
Figure 12. Menu Repo

Navigation Hierarchy

The navigation is built using the Amplience functionality called hierarchies. Below is a screenshot of the hierarchy that was implemented on the development system.

Figure 13. Expanded Navigation Hierarchy

Nav Root Node

The nav root node is the top of the hierarchy, which is then used to add the navigation items as either a Nav Menu Node or Single Nav Menu Node. This node does not get shown in the menu as it’s controlled by a checkbox flag called "Show in Menu?"

Schema - nav-root-node.json

Add node to the hierarchy:

  1. Put your mouse over the Root node and click on the 3 …​

  2. Select "Add child node"

    Figure 14. Add child node

  3. Choose to add either a "Nav Menu Node" or "Single Nav Menu Node"

  4. Choose "Add existing" to add an existing node or "Create and add new" to add a new item.

Figure 15. Add child node options

Only unique nodes are allowed to be added to a hierarchy as they cannot be used in more than one hierarchy.

Remove node from the Navigation

In this scenario the node to removed from the navigation is not removed from the hierarchy. The node is still valid, but isn’t always shown in the menu.

  1. Locate the node to be hidden in the hierarchy.

  2. Unselect the Show In Menu? flag and save.

  3. Publish the node.

Remove node from the Hierarchy

In this scenario the node should be completely removed from the hierarchy.

  1. Locate the node that is to be removed from the hierarchy and mouse over the node.

  2. Click the …​ to bring up a sub-menu.

  3. Select Remove node

    Figure 16. Remove node

  4. Once the node has been removed from the hierarchy go back to the Nav Hierarchy folder.

  5. In the upper right hand corner click on the List Settings Button and select to show Hierarchy Child nodes show the child nodes of the main hierarchy node.

    Figure 17. Show/Hide child nodes

  6. Locate the node that was removed, hover over the item and click on the …​ buttons at the right of the page to bring up a sub menu.

  7. Click on Publish to remove the relationship between the Nav Root Node and the child node.

Any nodes that are removed from the hierarchy have to be published after they have been removed in order to break the parent child relationship. If this step is not done the removed item will still be in the JSON output.

Footer Navigation

The footer is managed via the Amplience Hierarchy functionality, which is also how the main navigation is managed.

  3. Either type content page or search through the content until you find content page.

Scheduling content

Amplience allows for planning and scheduling content using a scheduling calendar interface, which uses the concepts of events and editions.

Events are used to represent major campaigns or project that have a start and end date. Within the events there can be one or more edition created that holds the content that needs to go live between the start and end dates defined in the Event.

Slots will be used to specify where the content should be delivered to for each page that uses slots. The homepage, PLP and responsiblity page types (exclusive offers) are pages that have slots defined that is known to the Front End PWA.

You can also schedule content without the use of slots for those pages that do not have slot definitions and aren’t using customer groups for the content. For example you could schedule a blog page to go live at a certain point on the site by building the event using content instead of slots.

When creating an event there are some things to consider before setting it up, which is listed below.

  • Which areas of the site is the content going to be created for.

  • What content is required.

  • When is the content scheduled to go live.

Creating an Event

Figure 23. Scheduling calendar

  1. Navigate to the Scheduling page by clicking on the Scheduling menu item.

  2. Use the Calendar to find the start date of the Event.

  3. Select the date and click on the Create Event button or right click and select create event.

    Figure 24. Create Event

  4. Give the event a name.

  5. Select the start and end date

  6. Add one or more locales for the event.

  7. Click "Save & Open event" to add editions and content to the event.

    Figure 25. Empty Event page

  8. Click on Create edition

    Figure 26. Create edition dialog

  9. Enter a name for the edition

  10. Select the start and end date for the edition.

  11. If you want the slot content to be empty once the edition ends click on the check box labeled "Slot content flagged to expire when edition ends".

  12. To create an edition based upon slots make sure that the Slots tab is selected and click on the link to "Add Slots".

    Figure 27. Add Slots

  13. Search the repository for the slot you want and drag it to the area on the left.

    Figure 28. Slot Repositories

  14. Select the slot and add content to the slot from the right hand side.

    Figure 29. Add slot information

  15. Repeat the steps to add as many slots to the edtion as needed.

  16. When done click on the "Schedule" button and you will see a confirmation box.

Figure 30. Schedule Confirmation

To preview the content on the site.

  1. Click on the Preview Button

    Figure 31. Preview confirmation

  2. Choose Quick preview or Date/Time Preview

  3. Choose the Preview Application.

  4. Click on Preview

  5. To use the Date/Time preview functionality the Edition needs to be Promoted to preview.

  6. To do this click on the Schedule drop down button and click on Promote to preview.
